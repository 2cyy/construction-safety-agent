# 广州腾讯总部大楼 — 施工安全管理 AI 智能体：理论分析与技术方案

---

## 一、场景定义

**场景**：广州腾讯总部大楼整栋硬装修工程，需要 AI 智能体 7×24 小时替代安全总监履行以下职责：

| 序号 | 业务需求 | 用户（安全总监）对 Agent 的典型提问 |
|------|----------|-------------------------------------|
| 1 | 实时识别工人未佩戴安全帽 | "现在 A 区有多少人没戴安全帽？""把刚才 3 楼没戴帽子的截图发我" |
| 2 | 实时识别工地吸烟行为 | "今天材料堆放区有没有人抽烟？""查一下最近一周所有抽烟违规记录" |
| 3 | 实时识别安全通道堵塞 | "3 楼东侧安全通道现在通不通？""昨天下午 4 点那条通道有没有被堵过？" |
| 4 | 每日下班前材料摆放合规检查 | "下班了，生成今天的材料摆放合规报告""钢筋堆放区和昨天比有没有退步？" |
| 5 | 综合安全态势查询与每日报告 | "今天整体安全评分多少？""对比上周，这周安全趋势是变好还是变差？" |

## 二、理论核心：商业问题 → 技术问题 → 解决方案

### 场景 1：实时安全帽佩戴检测

#### 1.1 用户可能产生的对话
- "看看 A 区 3 楼现在有多少人没戴安全帽？"
- "刚才从镜头 5 经过的那个蓝衣服工人，他戴安全帽了吗？"
- "今天累计安全帽违规多少次？主要集中在哪个区域？"

#### 1.2 商业问题 → 技术问题

| 商业问题 | 技术问题 | 难度 |
|----------|----------|------|
| 不同楼层光照差异大（地下室暗、高层强光） | **光照鲁棒性**：模型在极端光照下误检率高 | ⭐⭐⭐ |
| 工人可能被脚手架、管道遮挡 | **遮挡处理**：被遮挡 50%+ 时检测置信度骤降 | ⭐⭐⭐⭐ |
| 工人戴鸭舌帽/头巾与安全帽外观相似 | **细粒度分类**：安全帽 vs 鸭舌帽 vs 秃头 vs 头巾，类间差异小 | ⭐⭐⭐⭐ |
| 监控画面中远处工人安全帽仅几个像素 | **小目标检测**：安全帽在 1080P 画面中可能仅占 20×20 像素 | ⭐⭐⭐⭐⭐ |
| 多人密集通过同一区域时需要逐一识别 | **密集场景检测**：NMS 后处理在人群重叠时容易漏检 | ⭐⭐⭐ |

#### 1.3 技术解决方案

```
方案架构：
┌─────────────────────────────────────────────────────┐
│ 摄像头视频流 (RTSP)                                  │
│     ↓                                               │
│ 视频解码 + 抽帧 (每 2 秒 1 帧)                        │
│     ↓                                               │
│ YOLOv8s-Head 检测模型 (轻量化，适合边缘部署)           │
│   - 3 类检测：head（人头）、helmet（安全帽）、         │
│     other_hat（其他帽类）                             │
│     ↓                                               │
│ 逻辑判定层：                                          │
│   IF 检测到 head 但未检测到 helmet → 未戴安全帽        │
│   IF 检测到 other_hat 但未检测到 helmet → 疑似违规     │
│     ↓                                               │
│ DeepSORT 多目标追踪 → 给每个工人分配 ID               │
│     ↓                                               │
│ 违规事件生成 → 存入事件数据库                         │
│     ↓                                               │
│ 实时推送到 Agent 对话系统                             │
└─────────────────────────────────────────────────────┘
```

**关键解题点**：
- **小目标**：在 YOLOv8 颈部网络增加一层高分辨率特征图（P2 层），专门检测 <32×32 像素目标
- **光照**：训练数据加入 Albumentations 模拟的暗光/强光/逆光增强
- **遮挡**：使用 MixUp + CutOut 数据增强，模拟随机遮挡场景
- **密集**：将 NMS 替换为 Soft-NMS，保留重叠区域的高分候选框

### 场景 2：工地吸烟行为检测

#### 2.1 用户可能产生的对话
- "材料堆放区严禁烟火，现在有人在那抽烟吗？"
- "把今天所有抓到抽烟的截图和时间整理出来"
- "昨天有人在我警告之后还在抽，查出是谁了吗？"

#### 2.2 商业问题 → 技术问题

| 商业问题 | 技术问题 | 难度 |
|----------|----------|------|
| 香烟在画面中极小（1080P 中可能仅 5-8 像素） | **极小目标检测**：常规检测器感受野过大，对烟头无响应 | ⭐⭐⭐⭐⭐ |
| 手靠近嘴的动作也可能是喝水、擦汗、打电话 | **动作歧义**：单帧图像无法区分吸烟与喝水 | ⭐⭐⭐⭐ |
| 工人可能背对镜头吸烟，烟头不可见 | **视角盲区**：单摄像头无法覆盖 360° | ⭐⭐⭐⭐ |
| 工地通风好，烟雾快速扩散 | **烟雾特征弱**：视觉上几乎不可见 | ⭐⭐⭐⭐⭐ |

#### 2.3 技术解决方案

```
方案架构：
┌──────────────────────────────────────────────────┐
│ 方案 A（主路径）：视觉两阶段检测                    │
│                                                   │
│ 阶段 1 — 姿态估计（MediaPipe / YOLOv8-Pose）      │
│   检测手部关键点 + 嘴部关键点                      │
│   计算：手腕到嘴的距离 < 阈值 → 候选"手到口"动作    │
│     ↓                                             │
│ 阶段 2 — 烟头检测（放大 ROI 后 YOLOv8n-tiny）     │
│   在手-嘴区域裁剪放大 3× → 送入轻量检测器           │
│   检测：cigarette（烟头）、smoke_puff（烟团）       │
│     ↓                                             │
│ 时序确认（5 帧滑动窗口，≥3 帧阳性 → 确认违规）     │
│                                                   │
│ 方案 B（辅助路径）：热成像                          │
│   在材料堆放区等高风险区域加装红外热像仪            │
│   烟头温度 400-800°C → 热像仪上非常明显            │
│   几乎零误报，但成本较高                           │
└──────────────────────────────────────────────────┘
```

**关键解题点**：
- **极小目标**：不直接在原图上检测烟头，而是先定位手-嘴交互区域，在该区域放大后检测
- **动作歧义**：使用时序信息（连续 5 帧中有 3 帧阳性才判定），过滤喝水等短暂动作
- **视角盲区**：高风险区域双摄像头对角覆盖

### 场景 3：安全通道堵塞检测

#### 3.1 用户可能产生的对话
- "3 楼东侧安全通道现在通不通？"
- "过去 24 小时这条通道被堵过几次？每次堵了多久？"
- "如果有通道被堵超过 10 分钟，立即通知我"

#### 3.2 商业问题 → 技术问题

| 商业问题 | 技术问题 | 难度 |
|----------|----------|------|
| "堵塞"定义模糊 — 放一个桶算不算堵？堵一半算不算？ | **阈值定义**：需要量化的堵塞判定标准 | ⭐⭐⭐ |
| 工人临时放下一堆材料 2 分钟后拿走，这要不要报？ | **临时/永久区分**：短时放置 vs 真正堵塞 | ⭐⭐⭐⭐ |
| 不同楼层安全通道位置不同，需要分别配置 | **ROI 标定**：每条通道需要单独标注监测区域 | ⭐⭐ |
| 通道中有人站着聊天也算占用，但不等于"堵塞" | **人/物区分**：行人占用 vs 物体堵塞 | ⭐⭐⭐ |

#### 3.3 技术解决方案

```
方案架构：
┌──────────────────────────────────────────────────┐
│ 初始化阶段（每条安全通道执行一次）：                │
│   人工标注通道 ROI 多边形区域                      │
│   采集"空通道"参考帧 → 背景建模                    │
│     ↓                                             │
│ 实时监测阶段：                                     │
│   逐帧计算 ROI 内前景占比                          │
│   前景分割：SAM (Segment Anything) 或传统 MOG2     │
│     ↓                                             │
│ 判定逻辑：                                         │
│   IF 非人前景占比 > 30% AND 持续 > 5 分钟：       │
│     → 黄色预警（临时堵塞）                         │
│   IF 非人前景占比 > 60% AND 持续 > 1 分钟：       │
│     → 红色警报（严重堵塞）                         │
│   IF 非人前景占比 > 30% AND 持续 > 30 分钟：      │
│     → 升级为红色警报                               │
│     ↓                                             │
│ 事件推送至 Agent，包含：通道编号、堵塞程度、        │
│   持续时间、实时截图                               │
└──────────────────────────────────────────────────┘
```

**关键解题点**：
- **阈值模糊**：引入三级预警机制（绿/黄/红），根据占用比例 + 持续时间联合判定
- **临时/永久**：设置"持续时间"维度，< 5 分钟不计入堵塞事件
- **人/物区分**：使用 YOLOv8 行人检测排除站立的人，只计算无人物体占据面积

### 场景 4：每日材料摆放合规检查

#### 4.1 用户可能产生的对话
- "下班了，帮我检查所有材料区是否合规"
- "钢筋区的摆放和标准比差了多少？"
- "本周材料摆放合规率趋势如何？"

#### 4.2 商业问题 → 技术问题

| 商业问题 | 技术问题 | 难度 |
|----------|----------|------|
| "合规摆放"对不同材料有不同标准 | **多标准建模**：钢筋、木板、管道各有摆放规范，无法用统一规则判断 | ⭐⭐⭐⭐ |
| 材料区每天使用后状态不同，没有固定"空状态" | **动态基准**：无法用简单的背景减法，因为合规状态本身就是"有材料" | ⭐⭐⭐⭐ |
| 需要量化"合规程度"而非简单二分类 | **连续评分**：需要输出 0-100% 合规度而不是"合规/不合规" | ⭐⭐⭐ |
| 下班前检查时可能光线已变暗 | **光照变化**：黄昏/灯光下与白天差异大 | ⭐⭐⭐ |

#### 4.3 技术解决方案

```
方案架构：
┌──────────────────────────────────────────────────┐
│ 初始化阶段（每个材料区执行一次）：                  │
│   1. 拍摄"标准合规摆放"照片（人工摆好后采集）      │
│   2. 使用 MobileSAM 提取参考图像中各材料 mask      │
│   3. 存储为"合规模板"：{区域 ROI, 参考特征向量}    │
│     ↓                                             │
│ 每日检查阶段（如下午 17:30 触发）：                 │
│   1. 采集当前帧                                   │
│   2. 使用 MobileSAM 提取当前 mask                  │
│   3. 计算与参考模板的结构相似度（SSIM）             │
│   4. 对比关键特征：                                │
│      - 材料是否在指定区域内？(IoU 对比)            │
│      - 堆放是否整齐？(边缘直线度检测)              │
│      - 是否有散落材料？(区域外前景检测)            │
│   5. 综合评分 → 合规度 0-100%                     │
│     ↓                                             │
│ 报告生成：                                        │
│   每个材料区合规度 + 不合格区域标注 + 对比截图      │
└──────────────────────────────────────────────────┘
```

**关键解题点**：
- **多标准**：每个材料区存储独立的"合规模板"，而非全局规则
- **动态基准**：使用 SSIM（结构相似度）+ 边缘直线度（Canny + Hough）量化"整齐程度"
- **连续评分**：综合多个指标的加权评分公式：`Score = 0.3×SSIM + 0.2×区域IoU + 0.3×整齐度 + 0.2×散落度`

### 场景 5：自然语言交互 & 每日报告（Agent 核心）

#### 5.1 用户可能产生的对话
- "今天整体情况怎么样？"
- "对比上周三，今天安全帽违规是多了还是少了？"
- "给我生成这周的安全周报，重点突出抽烟问题"
- "有没有哪个区域最近 3 天违规持续上升？需要重点关注"

#### 5.2 商业问题 → 技术问题

| 商业问题 | 技术问题 | 难度 |
|----------|----------|------|
| 用户用自然语言提问，需要转化为结构化查询 | **NL2SQL / 意图解析**：将模糊自然语言映射到精确的数据查询 | ⭐⭐⭐⭐ |
| "最近""上周""这周"等时间表达需要准确解析 | **时间实体识别**：相对时间表达 → 绝对时间范围 | ⭐⭐⭐ |
| 回答需要综合多个检测模块的数据 | **多源数据融合**：安全帽 + 吸烟 + 通道 + 材料数据统一查询 | ⭐⭐⭐ |
| 用户可能有对话中没有明确的关键词 | **语义匹配与同义词扩展**：如"安全帽"也叫"头盔""帽子""头上的" | ⭐⭐⭐ |

#### 5.3 技术解决方案

```
方案架构：
┌──────────────────────────────────────────────────────┐
│                    Agent 对话系统                      │
│                                                       │
│  用户自然语言输入                                      │
│       ↓                                               │
│  ┌─────────────────────────────────────┐              │
│  │ LLM 意图解析层 (GPT-4o / Claude)      │              │
│  │  - 意图分类：查询 / 报告 / 报警 / 闲聊 │              │
│  │  - 实体提取：时间、地点、对象、指标    │              │
│  │  - 同义词扩展：安全帽→helmet          │              │
│  │  - 输出：结构化查询 JSON              │              │
│  └─────────────────────────────────────┘              │
│       ↓                                               │
│  ┌─────────────────────────────────────┐              │
│  │ 查询执行引擎                         │              │
│  │  {                                   │              │
│  │    "intent": "query_violations",      │              │
│  │    "type": "no_helmet",              │              │
│  │    "time_range": "2026-06-30",       │              │
│  │    "zone": "A-3F",                   │              │
│  │    "aggregation": "count_by_hour"    │              │
│  │  }                                   │              │
│  └─────────────────────────────────────┘              │
│       ↓                                               │
│  ┌─────────────────────────────────────┐              │
│  │ 检测数据库查询 (SQLite / PostgreSQL) │              │
│  │  - violations 表（安全帽/吸烟违规）   │              │
│  │  - channel_blocks 表（通道堵塞）     │              │
│  │  - material_checks 表（材料合规）     │              │
│  │  - workers 表（工人追踪）            │              │
│  └─────────────────────────────────────┘              │
│       ↓                                               │
│  ┌─────────────────────────────────────┐              │
│  │ 回答生成 (LLM)                       │              │
│  │  数据 + 自然语言模板 → 用户友好回答   │              │
│  │  + 可选：图表/截图/表格              │              │
│  └─────────────────────────────────────┘              │
│       ↓                                               │
│  返回给用户（文本 + 图片 + 数据卡片）                   │
└──────────────────────────────────────────────────────┘
```

**关键解题点**：
- **同义词匹配**：不在代码层做关键词匹配，而是用 LLM 做语义理解，Embedding + 向量检索处理专业术语映射
- **时间解析**：用 LLM + 当前时间戳解析相对时间："今天"→`2026-06-30`，"上周三"→`2026-06-24`
- **多源融合**：统一事件数据模型，所有检测结果存入统一的 `events` 表，消除数据孤岛

## 四、边缘-云端协同部署策略

| 层级 | 部署位置 | 承担任务 | 硬件 |
|------|----------|----------|------|
| **边缘层** | 每层楼弱电间 (边缘盒子) | 视频解码、实时推理（安全帽/吸烟/通道）、事件初筛 | Jetson Orin NX × 每层 1 台 |

## 七、深度专题：抽烟行为识别（多阶段 Agent 管道）

### 7.1 为什么单帧 YOLO 检测不到烟？

一张 1280×1707 的工地照片中：
- person（人）：数百到上千像素
- helmet（安全帽）：几十到上百像素
- cigarette（香烟）：**仅 10~20 个像素**

而且很多安全帽检测模型的类别只有 `person / helmet / head`，根本没有 `cigarette` 这个类别。不是"识别失败"，而是**压根不会识别**。

### 7.2 真实烟雾 vs 手里拿烟

| 场景 | 可见特征 | AI 检测难度 |
|------|---------|------------|
| 手拿香烟但不抽 | 极小烟头（10-20px），无烟雾 | ⭐⭐⭐⭐⭐ 极难 |
| 正在吸（有烟雾） | 烟头 + 半透明烟雾 | ⭐⭐⭐⭐ |
| 手靠近嘴 + 烟头 | 姿态 + 小目标 | ⭐⭐⭐ |

**核心认识：抽烟 ≠ 有烟雾。** 工业系统不能只检测 smoke。

### 7.3 商业问题 → 技术问题 → 解决方案（完整链路）

#### 商业问题
工人在禁烟区（尤其材料堆放区）吸烟，存在火灾安全隐患，需要实时识别和告警。

#### 技术问题 1：香烟目标过小（10-20px）
- **方案**：SAHI (Slicing Aided Hyper Inference) 切片推理 + 超分辨率增强（ESRGAN），将 ROI 放大 4× 后检测
- **方案**：使用高分辨率摄像头（4K）覆盖高风险区域（材料堆放区）

#### 技术问题 2：单帧无法判断"吸烟行为"
- **方案**：不检测 "smoke" 而检测 "smoking behavior"——手部姿态 + 烟头 + 时间窗口
- **五层检测管道**：

```
Layer 1: YOLO 检测人员 → person bbox
    ↓
Layer 2: MediaPipe/OpenPose 检测手部关键点 + 嘴部关键点
    ↓
Layer 3: YOLO-tiny 检测烟头（在手部 ROI 内放大检测）
    ↓
Layer 4: 姿态判断（手到嘴的距离 < 阈值 + 持续时间）
    ↓
Layer 5: ByteTrack 连续追踪 + 时间窗口（≥3秒持续 → Smoking）
```

#### 技术问题 3：手靠近嘴 ≠ 吸烟（喝水/擦汗/打电话）
- **方案**：时序行为分析，区分吸烟与喝水
  - 吸烟：手指间夹持 + 周期性手→嘴往返（吸入→呼出循环）
  - 喝水：手掌包裹 + 仰头 + 单手
  - 打电话：手贴耳朵，非嘴部
- **方案**：加入烟头检测作为必要判定条件

#### 技术问题 4：工人背对镜头时无法检测
- **方案**：高风险区域（B-1F 材料堆放区）双摄像头对角覆盖，减少死角
- **方案**：红外热成像辅助（烟头温度 400-800°C，热像仪上极为明显）

#### 技术问题 5：误报率高（饮料瓶/笔误判为烟）
- **方案**：YOLO + Pose + Rule Engine 多模型融合
- **方案**：规则引擎设定判断链——必须同时满足：person ✓ AND hand_near_mouth ✓ AND cigarette_detected ✓ AND duration ≥ 3s
- **方案**：人工复核机制——疑似事件标记为"需人工复核"，二次确认后计入违规

## 八、扩展场景：更多商业问题 → 技术问题 → 解决方案

### 场景 6：危险区域入侵检测

**商业问题**：工人误入吊装区/高空坠物区/临时禁入区

| 技术问题 | 解决方案 |
|----------|----------|
| 不同区域边界动态变化（今天禁入区明天开放） | 动态 ROI 配置：安全员通过 Web 界面实时标定禁区多边形，热更新到边缘推理 |
| 管理人员 vs 工人权限不同（安全员可以进） | 人脸识别 + 角色权限表：检测到入侵后先查人员身份，安全员/监理白名单过滤 |
| 吊臂运动范围与人员轨迹交叉预警 | 3D 空间建模 + 运动轨迹预测：预测吊臂 3 秒后位置与人员轨迹的交集 |

### 场景 7：高空作业安全带检测

**商业问题**：工人在 2m 以上高空作业未系安全带

| 技术问题 | 解决方案 |
|----------|----------|
| 安全带在图像中极细，与衣服纹理混淆 | 人体关键点检测 + 安全绳几何分析：检测腰/背关键点连接线的存在性 |
| 如何判断工人在"高空"？ | 深度估计（MiDaS/Depth Anything）+ 脚底与地面的相对高度计算 |
| 工人可能系了安全带但不挂挂钩 | 挂钩连接点检测：追踪安全带尾端挂钩是否连接到固定锚点 |

### 场景 8：夜间施工照明不足检测

**商业问题**：夜间施工照明不足导致安全隐患增加

| 技术问题 | 解决方案 |
|----------|----------|
| 如何量化"照明不足"？ | 图像平均亮度 + 局部对比度：将画面分为 16 格，统计低于亮度阈值的格子占比 |
| 红外摄像头画面与可见光检测模型不兼容 | 红外图像域适应：用 CycleGAN 将红外图转为伪可见光图再送入检测模型 |
| 夜间工人疲劳度增加 | 姿态分析 + 移动速度监测：步履缓慢、频繁停顿 → 疲劳预警 |

### 场景 9：施工机械与人员安全距离监测

**商业问题**：挖掘机/叉车/起重机与工人距离过近

| 技术问题 | 解决方案 |
|----------|----------|
| 3D 空间中的真实距离如何估算？ | 单目深度估计 + 相机标定：利用已知尺寸的参照物（如标准安全帽直径 30cm）估算像素与实际距离的换算 |
| 机械和人员可能在不同高度层 | 3D 检测（如 BEVFormer）+ 多层空间建模 |
| 机械操作员的视野盲区 | 360° 环视拼接 + 盲区人员检测 + 声光报警 |

### 场景 10：围栏/护栏完整性检测

**商业问题**：临边防护栏杆被拆除或损坏

| 技术问题 | 解决方案 |
|----------|----------|
| 护栏在画面中占比小，难以检测 | 固定位摄像头 + 背景参考帧对比：围栏区域 ROI 的前景变化检测 |
| 风吹/光照导致误报 | 连续帧确认 + 形态学滤波：只有持续超过 30 秒的变化才算破坏 |
| 夜间难以判断 | 红外补光 + 夜间参考帧 |

### 场景 11：多人聚集风险（踩踏/坍塌预防）

**商业问题**：某区域人员密度过高，存在楼板荷载超标或踩踏风险

| 技术问题 | 解决方案 |
|----------|----------|
| 如何精确统计小区域内人数？ | 密度估计（CSRNet/MCNN）+ 热力图：输出每平方米人数热力图 |
| 楼板最大荷载如何与人数关联？ | 荷载映射表：每层楼预设 max_persons/m² → 超标自动告警 |
| 人员密集但属于正常施工（如开会/集合） | 时间判断 + 区域类型：集合区（开放空间）vs 悬挑区（高风险），不同阈值 |

### 场景 12：电动工具使用安全检测

**商业问题**：切割机/电钻等工具使用不规范（未戴护目镜、未戴手套）

| 技术问题 | 解决方案 |
|----------|----------|
| 护目镜和手套在图像中特征弱 | 手部/眼部 ROI 放大检测 + 专用 PPE 检测模型（Construction-PPE 含 goggles/gloves 类） |
| 工人是否在"使用"工具而非"手持"？ | 动作识别：工具的振动/移动模式 + 手臂姿态判断使用状态 |
| 同一工人同时缺多项 PPE | 综合评分：单人安全得分 = f(helmet, goggles, gloves, vest, boots)，< 60 分告警 |

### 场景 13：临时用电安全检测

**商业问题**：配电箱未关闭、电缆拖地、私拉乱接

| 技术问题 | 解决方案 |
|----------|----------|
| 电缆在画面中极细，背景混淆 | 边缘检测 + 线状目标检测：Canny + HoughLinesP 检测地面线缆 |
| 配电箱门开/关状态判断 | 模板匹配：正常关闭状态 vs 当前状态的差异度 |
| 私拉乱接 vs 正常走线难以区分 | 合规线路预标定 + 新增线路检测：不在预标定路径内的线缆标记为疑似私拉 |

### 场景 14：粉尘/扬尘浓度预警

**商业问题**：切割/打磨产生大量粉尘，影响健康和可见度

| 技术问题 | 解决方案 |
|----------|----------|
| 可见光图像中粉尘特征弱 | 图像清晰度下降检测（Laplacian 方差降低 → 可见度下降 → 粉尘预警） |
| 粉尘 vs 水雾 vs 烟雾混淆 | 多模态：视觉 + PM2.5/PM10 传感器数据融合 |
| 粉尘来源定位 | 粒子扩散反向追踪 + 最近施工活动区域关联 |

### 场景 15：塔吊/升降机运行安全

**商业问题**：塔吊超载、升降机超员、恶劣天气下继续作业

| 技术问题 | 解决方案 |
|----------|----------|
| 摄像头无法直接测量重量 | 吊臂挠度视觉测量：通过检测吊臂弯曲程度反向推算载荷 |
| 升降机内人数统计 | 顶部摄像头 + 人头检测：Conf > 0.7 的人头计数 |
| 大风天气判断 | 图像中旗帜/树枝摆动幅度 + 气象 API 数据关联 |

### 场景 16：材料堆放稳定性检测

**商业问题**：管材/板材堆放过高或倾斜，有倾倒风险

| 技术问题 | 解决方案 |
|----------|----------|
| 如何判断"倾斜"？ | 边缘直线度检测：Canny + 霍夫变换提取材料边缘，计算与地面法线的夹角 |
| 如何判断"过高"？ | 参照物比例：以检测到的人员身高（平均 1.7m）为标尺，估算材料堆高度 |
| 倾斜角度随时间恶化 | 时序跟踪：每小时测量一次倾斜角，斜率 > 1°/h 触发预警 |

### 场景 17：废料/垃圾堆积检测

**商业问题**：施工废料未及时清理，堵塞通道且滋生安全隐患

| 技术问题 | 解决方案 |
|----------|----------|
| 废料 vs 在用料难以区分 | 区域类型标签 + 滞留时间：标记为"通道"的区域出现物体持续 > 2h = 废料 |
| 废料类型识别 | 语义分割 + 材质分类：木材/金属/塑料/混合垃圾 |
| 自动生成清理工单 | 检测到废料堆积 → 自动生成工单 → 推送到清洁班组 APP |

### 场景 18：应急设备可用性检测

**商业问题**：灭火器被遮挡、消防栓被材料挡住、应急灯不亮

| 技术问题 | 解决方案 |
|----------|----------|
| 灭火器/消防栓被遮挡检测 | 预标定设备位置 + 前景占用率：设备 ROI 区域被其他物体覆盖率 > 50% → 告警 |

| 设备巡检自动化 | 每日定时对所有预标定应急设备 ROI 做一次完整性检查，生成巡检报告 |

### 场景 19：噪音超标检测（融合 AI）

### 场景 20：工人身份与工时管理

### 场景 21：恶劣天气施工安全

| 技术问题 | 解决方案 |

### 场景 22：多摄像头跨区域事件关联

**商业问题**：同一工人在 A 区违规后走到 B 区，需要跨镜追踪

| 技术问题 | 解决方案 |
|----------|----------|
| 跨摄像头同一人识别（ReID） | DeepSORT + OSNet 行人重识别特征 + 时空约束（楼层拓扑） |

### 场景 23：安全交底/班前会检测

**商业问题**：工人未参加班前安全交底会就进入工地

| 技术问题 | 解决方案 |
|----------|----------|
| 如何知道一个工人"参加了交底会"？ | 人脸/安全帽 ID 签到 + 入场时间匹配：入场时间 < 交底会结束时间 → 违规 |
| 交底会区域人员统计 | 人头计数 + 持续时间判断（站立聚集 > 5min 才算参会） |

### 场景 24：隐患整改闭环追踪

**商业问题**：检测到隐患后，是否整改？何时整改？整改是否合格？

| 技术问题 | 解决方案 |
|----------|----------|
| 如何"追踪"一个隐患从发现到整改完成？ | 事件生命周期管理：detected → acknowledged → in_progress → resolved → verified |
| AI 如何自动验证整改是否合格？ | 整改前后同 ROI 区域对比：通道堵塞事件 → 24h 后自动拍摄同位置，占用率 < 10% = 整改合格 |
| 超期未整改自动升级 | SLA 规则引擎：warning 事件 4h 未处理 → 升级 critical → 推送项目经理 |

### 场景 25：安全趋势预测与主动预警

**商业问题**：不是等违规发生再告警，而是预测高风险时段/区域

| 技术问题 | 解决方案 |
|----------|----------|
| 如何从历史数据中找出规律？ | 时序分析（Prophet/LSTM）：按小时/天/周维度统计违规频率，识别周期性模式 |
| 如何预测"今天下午 A-3F 大概率出事"？ | 多特征输入：工人数量 + 施工类型 + 天气 + 历史违规率 → 风险预测模型 |
| 预测不准如何避免"狼来了"？ | 概率输出 + 分级推送：risk > 70% → 提醒安全员关注，risk > 90% → 强制增加巡检 |
